# Getting started — Operators (supervisors & field staff)

## Purpose

This guide explains how any **operator** (supervisor or aggregation field role) uses the GGEM Farming Dashboard to complete assigned checklists.

---

## Before you start

- You need a **Google account** (or email) registered in the system by an admin.  
- A **phone or computer** with internet.  
- For aggregation market days: know your **hub** and confirm a **session** was opened (see [SOP — Aggregation](./sop-aggregation.md)).

---

## Step 1 — Log in

1. Open the dashboard URL provided by your manager (hosted web app).  
2. Sign in with your assigned account.  
3. If you see **Unauthorized**, your role or department is not set correctly — contact admin.

You should land on **My Tasks** (supervisor / field portal).

---

## Step 2 — Understand your task list

On **My Tasks** you will see cards or rows for work assigned to you, for example:

- Checklist name (e.g. Milling, Weighing & Recording)  
- Status: pending, in progress, completed  
- Due date / shift if set by manager  

Tap or click a task to open the **checklist form**.

---

## Step 3 — Complete a checklist

### Navigation

- Checklists are split into **sections** (e.g. Section 1 — Setup).  
- Complete required fields (marked with * where configured).  
- Use **Next** / section list to move through the form.  
- Progress may save automatically while **in progress**.

### Common field types

| Type | What to do |
|------|------------|
| Text / number | Type the value |
| Checkbox | Tick when true |
| Select | Choose from dropdown |
| Log table | Use **Add row** for each farmer, bag line, milling run, etc. |
| Farmer search (aggregation) | Type 2+ letters of name, pick from list (see aggregation SOP) |
| Summary | Read-only totals — verify before submit |

### Location

If prompted, allow location when the checklist requires **on-site** verification.

### Submit

1. Review all sections.  
2. Use **Submit** (or final submit button) on the last section.  
3. Wait for confirmation — do not close the browser until you see success.  
4. The task should move to **completed**; a **submission** is created for manager review.

---

## Step 4 — View your past work

Open **My Submissions** (menu) to see forms you already submitted, status, and dates.

---

## Aggregation-specific reminder

If you work on **aggregation** checklists:

1. **Pre-Aggregation Setup** usually **opens the session** for the hub.  
2. All other aggregation forms must use the **same session** (linked automatically when configured).  
3. Complete forms in the order your manager defines: QC → Weighing → Warehouse receiving → End of day.

Details: [SOP — Aggregation](./sop-aggregation.md).

---

## Verification checklist (operator)

- [ ] I can log in and see my tasks  
- [ ] I can open a task and save progress  
- [ ] I can submit without errors  
- [ ] My submission appears under My Submissions  
- [ ] For aggregation: session ID on my forms matches today’s hub session  

---

## Troubleshooting

| Problem | What to try |
|---------|-------------|
| No tasks listed | Ask manager to assign a task in Task Management |
| Submit button disabled | Fill all required fields; check red validation messages |
| Farmer search empty / error | Need internet; wait a few seconds for list to load; type at least 2 letters |
| “Session not found” (aggregation) | Complete Pre-Aggregation Setup first or ask hub coordinator |
| Lost progress | Re-open same task ID; do not start a duplicate task unless manager says so |
