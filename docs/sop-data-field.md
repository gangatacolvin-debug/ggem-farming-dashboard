# SOP — Data and Field

**Department:** Data and Field  
**Applies to:** Field supervisors, data & field managers, leadership  
**Document type:** Standard Operating Procedure  

---

## 1. Purpose

Standardize how outreach, sales/marketing activities, field monitoring/QA, and call centre oversight are recorded in the GGEM Farming Dashboard.

---

## 2. Scope

| Checklist | Typical use |
|-----------|-------------|
| Outreach & Engagement | Farmer/community visits, events |
| Sales & Marketing | Campaigns, leads, promotions |
| Field Monitoring & QA | Plot visits, quality observations |
| Data / Call Centre Oversight | Call centre metrics and issues |

---

## 3. Roles and responsibilities

| Role | Responsibility |
|------|----------------|
| **Supervisor** | Complete assigned field tasks accurately and on schedule |
| **Manager** | Assign tasks by area/team; review submissions |
| **Leadership** | Department metrics and live task visibility |

---

## 4. Prerequisites

- Supervisor account with **data-field** department  
- Manager-created tasks with correct **scheduled date** and checklist type  

---

## 5. Standard procedure — Supervisor

### 5.1 Daily planning

1. Log in → **My Tasks**.  
2. Review tasks for today (outreach, monitoring, etc.).  
3. Prioritize by manager instructions (region, farmer group, deadline).

### 5.2 Completing outreach / monitoring forms

1. Open the correct checklist — name on card must match activity.  
2. Complete each section before leaving the field where possible (connectivity).  
3. Use log tables for multiple farmers, visits, or observations (**Add row** per entry).  
4. Attach accurate location when required.  
5. Submit before end of day unless task spans multiple days per manager.

### 5.3 Call centre oversight (if assigned)

1. Enter volumes, issues, and escalations in defined fields.  
2. Submit on the cadence agreed with manager (daily/weekly).

### 5.4 Handover

1. Confirm submissions in **My Submissions**.  
2. Brief manager on incomplete tasks or blocked fields.

---

## 6. Standard procedure — Manager

1. **Task Management** — assign checklist to supervisor with date.  
2. **Submissions Review** — approve/reject; request corrections via offline process if reject.  
3. **Live Monitoring** — optional check on in-progress field work.  
4. **Reports** — periodic review as configured.

---

## 7. Standard procedure — Leadership

1. Leadership dashboard → **Data and Field** department.  
2. **Live tasks** — see active field work.  
3. **Metrics & trends** — submission counts and lists by checklist type.

---

## 8. Verification

- [ ] Each planned visit/campaign has a corresponding submission for the period  
- [ ] Log table rows match paper notes or call logs  
- [ ] Manager approval completed per local SLA  

---

## 9. Exceptions

| Situation | Action |
|-----------|--------|
| No connectivity in field | Complete offline notes; submit when back on network (do not duplicate tasks) |
| Wrong checklist type | Contact manager before submit |
| Sensitive farmer data | Follow GGEM data privacy policy; only enter required fields |
